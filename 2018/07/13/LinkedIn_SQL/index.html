LinkedIn DS Phone Interview

Problem Statement: Member can make purchase via either mobile or desktop platform. Using the following data table to determine the total number of member and revenue for mobile_only, desktop_only and mobile_desktop.

The input spending table is:
|member_id| date| channel| spend|
|:——: |:——:| :——–:|:——:|
|1001| 1/1/2018| mobile| 100|
|1001| 1/1/2018| desktop| 100|
|1002| 1/1/2018| mobile| 100|
|1002| 1/2/2018| mobile| 100|
|1003| 1/1/2018| desktop| 100|
|1003| 1/2/2018| desktop| 100|

The output data is:
|date |channel |total_spend |total_members|
|:——: |:——:| :———:|:———–:|
|1/1/2018 |desktop |100 |1|
|1/1/2018 |mobile |100 |1|
|1/1/2018 |both |200 |1|
|1/2/2018 |desktop |100 |1|
|1/2/2018 |mobile |100 |1|
|1/2/2018 |both |0 |0|

  • 方法：看到output里面有both，明显channel这个column是用SELECT CASE做出来的，其他column都是直接groupby后aggregate即可得到。那么继续想将这个record CASE成mobile_only的条件就是每个member_id和date中没有desktop存在，也就是groupby member_id和date后SUM(IF(channel = 'desktop', 1, 0)) = 0。desktop_only的情况同理，both则是剩余所有情况。
  • TIPS: 这里用到了CONCAT(id, date)来做每笔交易的unique identifier，同时还用CAST(column AS char)来确保concat时两个变量都是string
